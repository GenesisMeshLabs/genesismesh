# Genesis Mesh Maintainer Plans

This directory contains maintainer-facing planning material for Genesis Mesh.
It is part of the repository so future maintainers can understand why releases
were shaped the way they were.

These files are not the public release source of truth. For shipped behavior,
use `CHANGELOG.md`, GitHub Releases, and the documentation under `docs/`.

## Files

- [`strategy.md`](strategy.md) - architectural thesis and long-term direction.
- [`roadmap.md`](roadmap.md) - milestone sequence and current planning context.
- [`go-to-market.md`](go-to-market.md) - adoption hypothesis and community
  wedge.
- [`plan-v0.1.0.md`](plan-v0.1.0.md) through
  [`plan-v0.21.1.md`](plan-v0.21.1.md) - release plans and retrospective
  release notes.

## Reading Notes

- Historical plans are retrospective when they were written after the tag.
- Future plans are directional and may change before release.
- Checkboxes in historical plans describe shipped or intentionally moved scope.
- A release tag does not inherit promises from a future plan.
